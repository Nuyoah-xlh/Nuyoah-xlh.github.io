SwitchHosts加速GitHub访问

hosts文件作用

​ hosts文件是用于本地dns服务的，采用ip域名的格式写在一个文本文件当中，hosts是一个没有扩展名的系统文件，可以用记事本等工具打开，其作用就是将一些常用的网址域名与其对应的IP地址建立一个关联“数据库”，当用户在浏览器中输入一个需要登录的网址时，系统会首先自动从Hosts文件中寻找对应的IP地址，一旦找到，系统会立即打开对应网页，如果没有找到，则系统再会将网址提交DNS域名解析服务器进行IP地址的解析。

SwitchHosts加速原理

​ 上文提到了hosts文件的作用，SwitchHosts就是通过自动修改hosts文件的内容，将访问GitHub最快的ip地址写入文件，当访问GitHub官网时，从hosts文件取到最优的地址直接访问，而不是通过层层域名服务器解析后再访问，这样能够加快域名解析速度，达到加速的作用。除了GitHub访问，同样也可以实现其他网站的快速解析加速。

​ SwitchHosts的一个重要功能是它能够与hosts文件关联，你可以设置一个定时间隔，每过一定时间，自动获取最新的最优ip访问地址并写入hosts文件，不必手动操作，省时省力。

具体步骤

下载SwitchHosts

下载地址：https://github.com/oldj/SwitchHosts/releases

根据需求下载后，直接安装即可。

添加hosts

安装完成后，用管理员身份运行，按照下图顺序添加hosts

Hosts类型选择远程

Hosts标题自行填写

URL填写 https://cdn.jsdelivr.net/gh/ineo6/hosts/hosts

自动刷新时间自行选择，建议选择1小时，最后确定即可。

开启加速

开启后，系统hosts会写入上图右边的数据，就算成功了。

注意

​ 如果显示访问权限不足无法开启，则应检查hosts的属性中权限是否开启，修改控制权限访问方法可自行百度，然后使用管理员身份运行即可。
